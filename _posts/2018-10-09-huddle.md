A while back I was working on trying to group together short audio clips using a
clustering technique that I called "Adversarial X-Means Clustering": an
extension of [x-means clustering][0]. The idea is fun but not yet fully formed
(I may write a post on it in the future) but as a throw away "lets try this
today" exercise I decided to give something else a go.

## Enter the penguin
![penguin](/assets/huddle/excuse-me-do-you-have-a-moment-to-talk-about-linux-penguin.jpg)

Instead of using fancy ML algorithms and being all theoretical about things, I
decided to take some advice from the fantastic [Jake Vanderplas][1]
and get a bit more hacky. To that extent, I ditched the mathematics and instead
went for a simple heuristic approach for grouping together data by "huddling"
waveforms together.

## Moving Earth
If you close your eyes and squint, you can treat almost any numeric dataset like
a histogram. It's not always a good idea to do so but in this case things work
out surprisingly well. A previous attempt to find a heuristic for comparing
discrete distributions had introduced me to the concept of the [Earth Mover's
Distance](https://en.wikipedia.org/wiki/Earth_mover%27s_distance), which is
almost exactly as it sounds: a measure of how hard it would be to turn one
"pile of earth" (distribution) into another.

My attempt at implementing this idea in code looks like this:
```python
def emd(v1, v2):
    '''Compute the Earth Mover Distance between two vectors.'''
    def _emd(ix):
        if ix == 0:
            return 0
        return abs(v1[ix] + _emd(ix-1) - v2[ix])

    distance = 0

    # Check for unequal length vectors
    l_v1, l_v2 = len(v1), len(v2)
    max_len = l_v1

    # "Move" all of the excess from the larger vector bailing if the lengths are
    # too different by an arbitrary amount (1.4 came from some manual testing).
    ratio = l_v1 / l_v2
    if ratio > 1.4 or ratio < (10 / 14):
        return sys.maxsize

    if l_v1 > l_v2:
        max_len = l_v2
        distance += sum(v1[l_v2:])

    elif l_v1 < l_v2:
        distance += sum(v2[l_v1:])

    distance += sum(_emd(ix) for ix in range(max_len))
    return distance / max_len
```

Essentially, backfill one distribution into the other and track the difference
in bin heights as we go. Again, pretty hacky but it gets the job done.

## Loading Some Data
To keep things simple, I dumped the audio in question as a CSV of floating point
values (one row per file) so that we can read the data into [pandas][2]. Also,
since it always take me an age to remember how to plot things neatly in a grid
using matplotlib, I've included the code for visualising the samples:
```python
import pandas as pd
import matplotlib.pyplot as plt


def get_samples(csv_sample_file):
    '''Open a CSV file of audio samples and convert to pandas series'''
    samples = []
    with open(csv_sample_file, 'r') as f:
        for line in f:
            samples = eval('[' + line + ']')
            samples.append(pd.Series(samples))

    return samples


def show_samples_grid(samples):
```

# And Now For The Fun Bit
You can hopefully see by eye that there are some obvious repeated patterns in
the samples shown above. So what we'd really like is to perform that same "yeah
they look about the same" style grouping just by looking at the data. I briefly
considered loading up [OpenCV][3] and doing this visually but instead I came up
with the following:

1) Determine the Earth Mover's Distance between each pair of files (horrible
exponential running time).<br>
2) If the distance between two files is less that a given threshold then count
those files as being "huddled" together and being part of the same group.<br>
3) Combine together smaller groups based on shared membership.<br>
4) Plot the resulting groups and see how we did.

In code it looks like this:
```python
def huddle_up(samples, threshold=int(2.4e7)):
    '''Attempt to group the samples'''
    emds = []

    for i in range(len(samples)):
        emds.extend([
            (i, j, emd(samples[i], samples[j]))
            for j in range(i+1, len(samples))
        ])

    sets = []

    # Merge sets that have common elements
    for l, r, dist in sorted(emds, key=lambda e: e[2]):
        if dist > threshold:
            break
        combined = {l, r}
        new_sets = []
        for s in sets:
            if l in s or r in s:
                combined.update(s)
            else:
                new_sets.append(s)

        sets = new_sets
        sets.append(combined)

    # Visualise the sets
    fig = plt.figure()
    for ix, S in enumerate(sets):
        series = [samples[ix] for ix in S]
        ax = plt.subplot(len(sets), 1, ix+1)
        for s in series:
            s.plot(figsize=(20, 8), ax=ax)
            ax.set_yticks([])
            ax.set_xticks([])
    fig.tight_layout()
    plt.show()
```
